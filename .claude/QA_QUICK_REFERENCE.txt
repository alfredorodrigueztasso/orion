TASK: Commit code changes
──────────────────────────
$ git add .
$ git commit -m "feat: new feature"

What happens:
  1. .husky/pre-commit runs
  2. validate-file-extensions.js checks (3s)
  3. validate-ssr-defaults.js checks (2s)
  4. validate-token-handlers.js checks (2s)
  If ❌ FAIL: Error message shown, commit blocked
  If ✅ PASS: Commit succeeds

Fix if fails:
  - Read error message carefully (it tells you what to fix)
  - Make the fix (e.g., rename .ts → .tsx)
  - git add the fix
  - git commit again


TASK: Build project
──────────────────────────
$ npm run build

What happens:
  1. npm run type-check (TypeScript)
  2. npm run build (Vite compiles to dist/)
  3. npm run validate (token check)
  4. validate-css-artifacts.js (CSS quality)
  If ❌ FAIL: Build stops, error shown
  If ✅ PASS: dist/ folder is ready

Fix if fails:
  - Read error message
  - Fix the TypeScript/build issue
  - npm run build again


TASK: Release new version
──────────────────────────
$ npm run release:patch    # for v1.0.0 → v1.0.1
$ npm run release:minor    # for v1.0.0 → v1.1.0
$ npm run release:major    # for v1.0.0 → v2.0.0

What happens:
  1. npm run prepublish:check (all 8 validations)
  2. Bump version in package.json
  3. npm publish --access public
  4. Create git tag v1.0.1
  If ❌ FAIL at any step: Release aborted, no publish
  If ✅ PASS all: Package on npm registry

Fix if fails:
  - Read detailed error from validation script
  - Fix the root cause (not a band-aid)
  - npm run release:patch again

Dry run (no publish):
$ npm run release:dry       # Test without publishing
