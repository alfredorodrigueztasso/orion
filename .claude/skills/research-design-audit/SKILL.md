# 📋 Research — Design Audit

Conduct a comprehensive UX audit of a feature, flow, or system. Analyze consistency, error handling, accessibility, and user friction points.

## Audit Dimensions

1. **Consistency** — Visual/interaction consistency across the feature
2. **Error Handling** — How are user errors handled?
3. **Feedback** — Is user always informed about what's happening?
4. **Accessibility** — Is it usable by all (keyboard, screen readers, color blind)?
5. **Performance** — Is it fast? Does it feel responsive?
6. **Completeness** — Are all edge cases handled?
7. **Mental Model** — Does it match user expectations?
8. **Delight** — Are there moments that surprise and delight?

## Output Deliverable

A Design Audit Report:
- Current state assessment
- Issues grouped by severity
- Consistency gaps
- Accessibility gaps
- Recommendations prioritized by impact
- Quick wins vs strategic improvements

Example structure:
- Consistency Issues (3 found)
- Accessibility Gaps (2 critical, 1 minor)
- Error Handling (incomplete in 2 scenarios)
- Performance Issues (1 slow interaction)
- Missing Confirmations (2 critical actions)
- Delight Opportunities (3 ideas)
