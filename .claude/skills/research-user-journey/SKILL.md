# 🗺️ Research — User Journey Mapping

Map the user's journey through a feature or product. Identify key touchpoints, emotional moments, pain points, and opportunities for improvement.

## Output Deliverable

A comprehensive User Journey Map including:
- **Phases**: Major stages in user flow (discovery → setup → usage → completion)
- **Touchpoints**: Key interactions and decision points
- **Emotions**: User feelings at each stage (happy, frustrated, confused, delighted)
- **Pain Points**: Problems and obstacles encountered
- **Opportunities**: Where design can improve the experience
- **Insights**: Key findings and patterns

## Your Process

1. **Understand the Context**
   - Ask clarifying questions about user type, goal, context
   - Define journey scope and timeframe
   - Identify constraints or dependencies

2. **Map the Journey**
   - Break into distinct phases (3-5 stages)
   - Identify key actions and decisions
   - Note emotional highs and lows
   - Highlight pain points

3. **Synthesize Insights**
   - Pattern recognition across phases
   - Biggest frustration points
   - Biggest delighting moments
   - Design opportunities

4. **Deliver Results**
   - Structured journey map (table or visual)
   - Key insights summary
   - Top 3-5 opportunities for improvement

Example output:

```markdown
# User Journey Map: Uploading a Photo to Profile

## Phase 1: Discovery (Emotional Arc: Curious → Confident)
- **Action**: User opens settings looking for profile options
- **Emotion**: Curious ("Where's my profile?")
- **Pain Point**: Settings menu has 12 options, hard to find
- **Opportunity**: Highlight "Profile" with icon or badge

## Phase 2: Profile Page (Emotional Arc: Confident → Expecting)
- **Action**: User finds "Edit Profile" button
- **Emotion**: Confident ("Found it!")
- **Pain Point**: Button is small and low-contrast
- **Opportunity**: Make CTA button more prominent

## Phase 3: Photo Upload (Emotional Arc: Expecting → Frustrated → Delighted)
- **Action**: User clicks upload, waits for image process
- **Emotion**: Frustrated ("Why is this taking so long?")
- **Pain Point**: No loading indicator, no feedback
- **Opportunity**: Add progress bar and status message

- **Action**: Photo uploads successfully
- **Emotion**: Delighted ("Done!")
- **Pain Point**: No confirmation message
- **Opportunity**: Show success toast with undo option

## Top Opportunities
1. **Visibility**: Make profile editing more discoverable
2. **Feedback**: Add loading states and success confirmations
3. **Control**: Let users see upload progress
```

Use this to inform design decisions and identify high-impact improvements.
