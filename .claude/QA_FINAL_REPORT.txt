Pregunta inicial:
  "¿Por qué 4.9.2 se publicó a npm aunque estaba roto?"

Respuesta encontrada:
  El pipeline de release NO VALIDABA el contenido de dist/ antes de publicar.

                           HALLAZGOS PRINCIPALES
═══════════════════════════════════════════════════════════════════════════════

1. ¿QUÉ PASÓ EN LA COMPILACIÓN?
────────────────────────────────────
   npm run build:release ejecutó pero produjo output INCOMPLETO:
   
   ✓ packages/react/dist/theme.css (9.7 KB)
   ✗ packages/react/dist/index.mjs (MISSING)
   ✗ packages/react/dist/index.cjs (MISSING)
   ✗ packages/react/dist/index.d.ts (MISSING)
   ✗ Otros 19 archivos de las subpaths: blocks/, chart/, calendar/, etc. (MISSING)

2. ¿POR QUÉ NO SE DETECTÓ EL PROBLEMA?
──────────────────────────────────────
   scripts/release.js línea 239:
   
   const buildResult = exec('npm run build:release');
   if (!buildResult.success) {
     logError('Build failed.');
     process.exit(1);
   }
   
   PROBLEMA: Solo chequea exit code del proceso.
   Exit code fue 0 (éxito) aunque dist/ estaba incompleto.
   
   SIN VALIDACIÓN: Nunca se chequea el contenido de dist/

3. ¿POR QUÉ NPM PERMITIÓ PUBLICAR BASURA?
─────────────────────────────────────────
   npm no valida que package.json exports tenga files correspondientes.
   
   npm.org recibió un tarball donde:
   - package.json declara 22 exports
   - dist/ solo tiene 1 archivo (theme.css)
   - npm publicó como está (no hay validación)

4. ¿POR QUÉ TENEMOS validate-exports.js Y NO SE USA?
────────────────────────────────────────────────────
   scripts/validate-exports.js existe (creado 21 Mar 00:33)
   Hace exactamente lo que necesitábamos
   Pero release.js NUNCA LO LLAMA
   
   Tools existe pero está DESCONECTADO del pipeline

═══════════════════════════════════════════════════════════════════════════════
                        BRECHAS DE VALIDACIÓN ENCONTRADAS
═══════════════════════════════════════════════════════════════════════════════

BRECHA 1: Sin validación de output de build
  Ubicación: scripts/release.js línea 239
  Síntoma: Exit code 0 ≠ build completo
  Impacto: Builds incompletos se publican como exitosos
  Severidad: CRÍTICA

BRECHA 2: Herramienta de validación existente pero sin usar
  Ubicación: scripts/validate-exports.js (creado pero nunca usado)
  Síntoma: Script existe pero release.js no lo llama
  Impacto: Validación disponible pero inactiva
  Severidad: CRÍTICA

BRECHA 3: Sin chequeo de archivos mínimos
  Ubicación: release.js línea 239
  Síntoma: dist/ puede estar vacío o tener solo CSS
  Impacto: Builds parciales no son detectados
  Severidad: ALTA

BRECHA 4: Sin validación pre-publish
  Ubicación: release.js línea 264 (npm publish)
  Síntoma: npm publish se ejecuta sin inspeccionar tarball
  Impacto: Paquetes rotos en npm.org
  Severidad: MEDIA

BRECHA 5: Sin chequeo post-publish
  Ubicación: release.js línea 311 (fin)
  Síntoma: Después de publicar no se verifica instalabilidad
  Impacto: Regressions no detectadas hasta que usan el package
  Severidad: MEDIA
