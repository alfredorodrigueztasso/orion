2. release.js executes Step 4 (Build):

5. release.js publishes without checking what was ACTUALLY built:

     exec('npm publish --access public');  // Publishes incomplete tarball
   }

6. npm publishes the broken package AS-IS:
   - npm doesn't validate that declared exports exist
   - npm.org now has useless package
   - Users cannot use it

================================================================================
WHY THIS WASN'T CAUGHT
================================================================================

1. Build Validation Missing
   - release.js runs: npm run build:release
   - Checks: exit code only
   - Doesn't check: dist/ directory contents
   - Result: Incomplete builds are treated as successful

2. NPM's Permissive Publishing
   - npm doesn't validate that exports paths exist
   - npm publishes the tarball regardless of completeness
   - No automatic verification of declared vs actual exports

3. Existing Tools Not Integrated
   - validate-exports.js was created to catch this
   - But it's not called by release.js
   - No automation means it could be forgotten

4. No Post-Publish Verification
   - After npm publish, no test of actual installation
   - No verification that imports work from the registry
   - No automated regression detection

================================================================================
SOLUTION: Build Completeness Validation Script
================================================================================

New Script: scripts/validate-build-completeness.js

What it does:
  ✓ Checks dist/ exists and is not empty
  ✓ Verifies all exports declared in package.json have files in dist/
  ✓ Detects auto-numbered collision files (Vite multi-entry bug)
  ✓ Lists exact missing files with export paths
  ✓ Provides diagnostic hints for fixing build

Integration:
  1. Add to release.js Step 4 (after build:release)
  2. Add npm script: "validate:build"
  3. Release aborts if validation fails
