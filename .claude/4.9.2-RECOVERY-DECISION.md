# v4.9.2 Recovery Decision & Action Plan

**Date**: March 21, 2026
**Status**: DECISION MADE - Deprecate v4.9.2 + Publish v4.9.3

---

## The Problem

v4.9.2 was published to npm with **incomplete build output**:
- Only `dist/theme.css` exists (9.8 KB)
- **MISSING**: All main exports (index.mjs, index.cjs, index.d.ts)
- **MISSING**: All named exports (tokens/, sections/, blocks/, templates/)
- **MISSING**: Heavy component exports (chart, calendar, editor, dnd, rich)
- **MISSING**: Client export, styles.css, blocks.css

**Root Cause**: TypeScript build (`tsc`) failed on circular imports in tsconfig.build.json, preventing Vite from running.

**Release Process Gap**: No validation of dist/ completeness before npm publish.

---

## Decision Matrix

| Option | Pros | Cons | Recommendation |
|--------|------|------|---|
| **1. Deprecate v4.9.2** | Clear messaging to users; old installs still work; users see upgrade prompt | Users on 4.9.2 must manually upgrade | ✅ **CHOSEN** |
| **2. Yank (Unpublish)** | Prevents new installs; forces upgrade | Users with 4.9.2 locked still broken; npm shows "not found" (confusing) | ❌ Consider only if critical |
| **3. Leave as-is** | No action needed | Users get broken package; support burden; unclear path forward | ❌ Worst option |

### Why Deprecate?

1. **Transparency**: Users see `npm WARN deprecated ...` when installing
2. **Reversibility**: Can un-deprecate if needed; doesn't break existing installs
3. **Upgrade Path**: Clear message points users to 4.9.3
4. **User Control**: Users who locked 4.9.2 can still use it (at their risk)
5. **Support**: Reduces confusion - old installs still work, but flagged
6. **Precedent**: Standard practice in npm ecosystem

## Post-Mortem: Why This Happened

1. **Build Check Missing**: Released without verifying dist/ completeness
2. **Silent Failure**: `turbo run build` didn't bubble up tsc errors clearly
3. **No Validation**: No pre-publish check for exports vs files
4. **Process Gap**: Release script trusts build output without verification

**Prevented by**: validate-dist-completeness.js + test suite added to release pipeline
