## The Problem (For Reference)

**File**: `registry/preview-modules/index.ts`
**Content**: React components with JSX (`<Button>`, `<Card>`, etc.)
**Issue**: `.ts` files are treated as pure TypeScript (no JSX)
**Result**: Next.js build fails with "Expected '>', got 'style'" syntax error

---

## The Fix (What to do)

### Step 1: Rename File
```bash
cd /Users/alfredo/Documents/AI\ First\ DS\ Library
mv registry/preview-modules/index.ts registry/preview-modules/index.tsx
```

**Why**: `.tsx` extension tells TypeScript this file contains JSX


## Common Issues & Fixes

### Issue: "File still exists as index.ts"
**Cause**: Rename didn't work or cached
**Fix**:
```bash
rm registry/preview-modules/index.ts
ls registry/preview-modules/
# Should only show index.tsx
```

### Issue: "Build fails, still can't find dist files"
**Cause**: Previous dist/ not cleaned
**Fix**:
```bash
cd packages/react
rm -rf dist/
npm run build
```

### Issue: "Validation says exports missing"
**Cause**: Build didn't generate all files
**Fix**:
```bash
# Check which exports are missing
node scripts/validate-exports.js
# Verify Vite config has preserveModules: true
cat vite.config.ts | grep preserveModules
```

### Issue: "npm publish says already published"
**Cause**: Version already exists
**Fix**:
```bash
# Check what's on npm
npm view @orion-ds/react
# If 4.9.6 already exists, use 4.9.7 instead
npm version patch  # 4.9.6 → 4.9.7
```
