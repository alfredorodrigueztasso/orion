# Orion Design System

Orion is an **AI-first design system** built on the "Chain of Truth" principle. It's a token-governed framework that eliminates UI hallucination by enforcing strict separation between primitives, semantics, and components.

## The Chain of Truth

Orion's architecture is a 3-layer system where each layer has a single responsibility:

<CardGrid columns={3}>
  <FeatureCard icon="🧱" title="Primitives (Layer 1)">
    Raw values (colors, sizes, fonts) defined in <code>tokens/primary.json</code>. The immutable source of truth. E.g. <code>#1B5BFF</code>, <code>16px</code>.
  </FeatureCard>
  <FeatureCard icon="🎯" title="Semantics (Layer 2)">
    Intent-based aliases like <code>--surface-base</code>, <code>--text-primary</code>. Defined per theme (light/dark). Describe <em>what values do</em>, not what they look like.
  </FeatureCard>
  <FeatureCard icon="🧩" title="Components (Layer 3)">
    Blind consumers that reference semantic tokens via CSS variables. Components never hardcode values — they just work across any brand or theme.
  </FeatureCard>
</CardGrid>

This architecture guarantees that switching brands or themes updates the entire UI consistently — zero visual drift.

## Multi-Brand Support

## Tri-Modal System

Components adapt their visual behavior based on the active mode:

<CardGrid columns={3}>
  <FeatureCard icon="🍎" title="Display Mode">
    Atmospheric & Narrative (Apple-style). Generous shadows, -4px hover lifts, glassmorphism enabled. For marketing and landing pages.
  </FeatureCard>
  <FeatureCard icon="📐" title="Product Mode">
    Geometric Precision (Linear-style). Minimal shadows, color-change-only hover, no glassmorphism. For dashboards and admin panels.
  </FeatureCard>
  <FeatureCard icon="📱" title="App Mode">
    Tactile Elevation (Material 3-style). Native shadows, -2px hover lifts, touch-friendly. For mobile apps and touch interfaces.
  </FeatureCard>
</CardGrid>

Each mode controls shadows, hover lifts, glassmorphism, and spacing — automatically.
